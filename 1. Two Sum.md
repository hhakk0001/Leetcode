* 暴力法:
    * 時間複雜度: $O(n^2)$
    * 空間複雜度: $O(1)$
* Hash Table:
    * 時間複雜度: $O(n)$
    * 空間複雜度: $O(n)$

## 為什麼用雜湊表可以減少查找元素的時間

雜湊表（Hash Table）是根據「鍵（key）」來存取資料的一種資料結構，核心優勢在於：

每個鍵透過雜湊函式（hash function）計算後，對應到記憶體的某個位置（桶 bucket）

查找元素的時間複雜度理論上為 O(1)，也就是常數時間

| 結構  | 查找時間複雜度 | 範例說明               |
| --- | ------- | ------------------ |
| 陣列  | O(n)    | 需逐一比對每個元素是否為目標值    |
| 雜湊表 | O(1)    | 根據 key 直接對應到值所在的位置 |

## 為什麼索引是放在 key，而不是 value

一來通常字典內的映射關係是 `key -> value`，二來 `in` 運算子用於字典時，默認檢查的是鍵。

如果我們用:
```python
nums_dict[i] = nums[i]
```

那麼以值查詢索引就得用:
```python
for i, num in nums_dict.items():
    if num == complement:
        return [i, ...]

# 或是

if complement in nums_dict.value():
```

`.items()`跟`.value()` 本質上等價於線性搜尋(時間為 $O(n)$)，這樣使用雜湊表就失去了意義。
