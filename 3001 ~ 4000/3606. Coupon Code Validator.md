# 251213

偏簡單、注意排序之條件。

# 題目

給三個等長的陣列: 
* `code[i]`：第 i 張優惠券代碼（Coupon Code），字串。
* `businessLine[i]`：該優惠券對應的業務線（Business Line），字串。
* `isActive[i]`：該優惠券是否有效（Active），布林值。
  
要求:
* `code[i]` 為非空，且只包含英數或底線 `_`。
* `businessLine[i]` 必須是 "electronics", "grocery", "pharmacy", "restaurant" 其中之一。
* `isActive[i]` 為 true。

回傳符合資格的優惠券代碼之列表，其中答案須**按照業務線順序(由左到右)與優惠券代碼(字母順序)進行排序**後列出。

## 模擬法

篩選優惠券的操作並不難，重點在於如何將答案先後按照 `businessLine[i]` 優先度跟 `code[i]` 進行排序。

一種比較直觀的做法是直接把四種業務線的優惠券分開存入對應的 List，分別進行排序後再串接。  
其他大部分解法也大多是基於此概念，只是實作方式不同。

* 時間複雜度: $O(n \log n)$
* 空間複雜度: $O(k)$，k 為合法優惠券數。
